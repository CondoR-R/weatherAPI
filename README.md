# ⛅ Weather API

Rest API для получения данных о погоде с возможностью сохранения координат для получения почасового прогноза на сутки.  

## 🚀 Описание методов

### 1. Регистрация пользователя

**POST** `/users`

Принимает имя пользователя, если пользователь с таким именем есть в базе, возвращает его id, если пользователя с таким именем нет - создает его в базе и возвращает его id.

#### Тело запроса (JSON)

| Поле | Тип | Описание |
|------|-----|----------|
| `user_name` | string | Имя пользователя (обязательное) |


#### Пример использования

```
http://127.0.0.1:8000/users
с телом запроса (см. ниже)
```

#### Пример тела запроса (JSON)

```json
{
    "user_name": "string"
}
```

#### Пример ответа (JSON)

```json
{
    "id": 1
}
```

---

### 2. Получение текущей погоды

**GET** `/current_weather`  

Принимает координаты и возвращает данные о температуре, скорости ветра и атмосферном давлении на момент запроса.  

#### Параметры запроса (query string)

- **lat** – широта (число с плавающей точкой, от -90 до 90)
- **lon** – долгота (число с плавающей точкой, от -180 до 180)

#### Пример использования

```
http://127.0.0.1:8000/current_weather?lat=56.5&lon=85
```

#### Пример ответа (JSON)

```json
{
    "current_temperature": -28.44499969482422,
    "current_wind_speed": 1.0800000429153442,
    "atmospheric_pressure": 1043.4000244140625
}
```

---

### 3. Добавление города в список отслеживаемых городов  

**POST** `/cities`   

Принимает id пользователя, название города и его координаты и добавляет в список городов, для которых отслеживается прогноз погоды. После добавления города делает запрос к Open Meteo API, чтобы сразу получить почасовой прогноз.

#### Тело запроса (JSON)

| Поле | Тип | Описание |
|------|-----|----------|
| user_id |	integer |	ID пользователя (обязательное) |
| city | 	string | 	Название города (обязательное) |
| lat | 	float | 	Широта (от -90 до 90, обязательное) |
| lon | 	float | 	Долгота (от -180 до 180, обязательное) |

#### Пример использования

```
http://127.0.0.1:8000/cities
с телом запроса (см. ниже)
```

#### Пример тела запроса (JSON)

```json
{
    "user_id": 42,
    "city": "Москва",
    "lat": 55.7558,
    "lon": 37.6176
}
```

#### Пример ответа (JSON)

При успешном добавлении:

```json
{
    "detail": "City ​​added successfully"
}
```

В случае, если город с таким названием уже есть в базе для пользователя с переданным id:

```json
{
    "detail": "City already exists for this user"
}
```

---

### 4. Получение списка городов, для которых доступен прогноз погоды

**GET** `/cities`  

Принимает id пользователя для которого необходимо вывести список городов. Возвращает список городов, для которых отслеживается прогноз погоды.  

#### Параметры запроса (query string)

- **user_id** - id пользователя

#### Пример использования

```
http://127.0.0.1:8000/cities?user_id=1
```

#### Пример ответа (JSON)

```json
[
    "Томск",
    "Новосибирск",
    "Кемерово"
]
```

---

### 5. Получение почасового прогноза

**GET** `/weather`  

Принимает id пользователя, название города, время и набор возвращаемых параметров и возвращает для него погоду на текущий день в указанное время.

#### Параметры запроса (query string)

- **user_id** - id пользователя
- **city** - название города
- **time** - час, для которого необходимо вернуть прогноз погоды (от 0 до 23)
- **params**: возвращаемые параметры погоды, при отсутствии данного параметра 
    вернет все возможные параметры погоды. Параметры погоды записываются через
    запятую без пробела. Возможные параметры:  
    - **temperature**: температура
    - **relative_humidity**: относительная влажность
    - **wind_speed**: скорость ветра
    - **precipitation**: осадки

#### Пример использования

```
http://127.0.0.1:8000/weather?user_id=1&city=Томск&time=15&params=temperature,precipitation
```

#### Пример ответа (JSON)

```json
{
    "temperature": -23.645000457763672,
    "precipitation": 0.0
}
```

## ⚙️ Установка и запуск

Убедитесь, что у вас установлен **Python версии 3.10** или выше для корректной работы фреймворков и библиотек, используемых в проекте.

```bash
python --version
```

### 1. Клонирование репозитория

Склонируйте репозиторий с кодом и перейдите в его директорию:

```bash
git clone https://github.com/CondoR-R/weatherAPI.git
cd weatherAPI
```

### 2. Создание виртуального окружения (рекомендуется)

Виртуальное окружение позволит изолировать зависимости проекта от глобальных пакетов Python.  

**на Windows:**

```bash
python -m venv venv
venv\Scripts\activate
```
**на MasOS или Linux:**

```bash
python3 -m venv venv
source venv/bin/activate
```

После активации в терминале появится префикс `(venv)`.

### 3. Установка зависимостей

Все необходимые библиотеки перечислены в файле `requirements.txt`. Установите их с помощью команды `pip`:

```bash
pip install -r requirements.txt
```

### 4. Запуск приложения

Проект запускается командой:

```bash
python script.py
```

или:

```bash
python3 script.py
```

После запуска сервер будет доступен по адресу [http://127.0.0.1:8000](http://127.0.0.1:8000).  
Документация API (Swagger UI) откроется по ссылке [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs).

### Остановка серверва

Остановка осуществляется вводом в терминал комбинации клавиш `CTRL` + `C`

## 📝 Документация API

После запуска проекта автоматическая документация (Swagger UI) доступна по адресу:

- **Swagger UI**: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)
- **ReDoc**: [http://127.0.0.1:8000/redoc](http://127.0.0.1:8000/redoc)

Эти интерфейсы позволяют интерактивно тестировать все методы, просматривать ожидаемые параметры и модели ответов.

## 🧪 Тестирование

Проект использует pytest для автоматического тестирования. Все тесты находятся в директории tests/.  

Запуск тестов осуществляется через терминал командой:

```bash
pytest
```

Или:

```bash
pytest -v
```

Для более подробной информации.

## 🛠 Технологии

- **FastAPI** – высокопроизводительный веб-фреймворк для создания REST API.
- **Uvicorn** – ASGI-сервер для запуска приложения.
- **SQLAlchemy** – ORM (Object-Relational Mapping) для работы с базой данных SQLite.
- **Pydantic** – валидация данных, управление настройками и сериализация/десериализация JSON.
- **aiosqlite** – асинхронный драйвер для SQLite, обеспечивающий неблокирующие операции с БД.
- **Open-Meteo SDK** (`openmeteo_requests`, `openmeteo_sdk`) – клиент для получения погодных данных из открытого API Open-Meteo.
- **Pytest** – фреймворк для модульного тестирования, включая плагины `pytest-asyncio` и `pytest-mock` для тестирования асинхронного кода и подмены зависимостей.
- **AnyIO** – библиотека для работы с асинхронностью, обеспечивающая совместимость с разными event loop'ами (asyncio, trio).
- **Starlette** – лёгкий ASGI-фреймворк, на котором базируется FastAPI.

Все зависимости зафиксированы в файле `requirements.txt` с указанием точных версий для воспроизводимости окружения.
